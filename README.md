# Project & Task Management Platform

A full-stack Project & Task Management Platform with user authentication, project management, task assignment, status tracking, search, and AI-generated task summaries.

---

## Features

### Authentication
- User registration
- Login using JWT
- Protected API routes (Bearer token via `Authorization` header)

### Project Management
- Create, read, update, delete projects
- Ownership-scoped access (a user can only modify their own projects)
- Pagination and name search on the projects list

### Task Management
- Create tasks within a project
- Assign tasks to a registered user
- Update task status (`TODO`, `IN_PROGRESS`, `DONE`)
- Priority levels (`LOW`, `MEDIUM`, `HIGH`)
- Pagination and filtering (by status, priority, project) on the tasks list

### Search
- Full task search across title and description
- Scoped to tasks the requesting user created or is assigned to
- Paginated results

### AI Integration
- Google Gemini API integration for one-line AI-generated task summaries
- Graceful fallback to a rule-based summary if the AI call fails (missing key, rate limit, model deprecation, network error)

---

## Tech Stack

**Backend**
- Node.js + Express
- TypeScript
- Prisma ORM (Prisma 7)
- PostgreSQL
- Redis (rate limiting / caching)

**AI**
- Google Gemini API (`gemini-2.5-flash` by default, configurable)

**DevOps**
- Docker

## Local Development Setup

### 1. Clone the repository

```bash
git clone <repository-url>
cd backend
```

### 2. Install dependencies

```bash
npm install
```

### 3. Create your `.env` file

```env
DATABASE_URL=postgresql://user:password@localhost:5432/yourdbname
PORT=5000
CORS_ORIGIN=http://localhost:3000
JWT_SECRET=your_jwt_secret_here
REDIS_URL=redis://localhost:6379
GEMINI_API_KEY=your_gemini_api_key_here
GEMINI_MODEL=gemini-2.5-flash
```

> Get a free Gemini API key at [aistudio.google.com](https://aistudio.google.com) — no payment method required to start.
>
> **Note on `GEMINI_MODEL`:** Google deprecates Gemini model versions frequently. If the AI summary feature throws a 404, run:
> ```bash
> curl.exe "https://generativelanguage.googleapis.com/v1beta/models?key=YOUR_KEY"
> ```
> and pick any model from the response where `supportedGenerationMethods` includes `generateContent`.

### 4. Generate the Prisma client

```bash
npx prisma generate
```

### 5. Run database migrations

```bash
npx prisma migrate dev
```

### 6. Start the dev server

```bash
npm run dev
```

Backend runs at `http://localhost:5000`.

## Known Limitations / Future Improvements

- Automated unit and integration tests are not yet implemented — see `docs/testing.md` for details and rationale.
- No service-layer abstraction yet; controllers call Prisma directly. Fine at this scope, would extract as the codebase grows.
- Redis is wired up for rate limiting but not yet used for query result caching.
- WebSocket-based real-time status updates (currently requires a page refresh / re-fetch).
- File attachments on tasks.
- Activity log / audit trail.
